KaninDex Privacy Policy

Privacy Policy

Last updated: May 2, 2026

This page explains how KaninDex handles local project indexing, remote LLM provider usage, stored settings, and user control.

English

1. Overview

KaninDex is designed to help users understand AI-generated codebases by indexing project files, symbols, relationships, and explanatory summaries.

2. Data Processed by the App

KaninDex may process the following categories of information on-device:

  • Selected project files and folder structure
  • Generated index summaries, symbol descriptions, and relationship notes
  • User questions and conversation history
  • Provider settings needed to connect to local or remote LLM services

3. Local-First Behavior

KaninDex is designed with a local-first workflow. Local indexing, cached project structure, and many browsing functions can work without sending content to a remote service.

4. Remote Processing

If the user chooses a remote LLM provider, portions of project context may be transmitted to that provider in order to answer questions or generate explanations. This may include:

  • User prompts
  • Project summaries
  • Relevant file, symbol, and relationship context
  • Code excerpts needed to generate a response

KaninDex presents a consent step before this remote processing path is used.

5. Credentials and Stored Settings

Provider settings, project indexes, and conversation history may be stored locally on the device. Secrets and OAuth tokens are intended to be stored in encrypted form when available.

6. User Control

Users can:

  • Choose whether to use remote providers
  • Decline remote processing consent
  • Remove indexed projects and local conversation history

7. Contact

For privacy-related questions, contact shteosis@gmail.com.

한국어

1. 개요

KaninDex는 AI가 생성한 코드베이스를 이해할 수 있도록 프로젝트 파일, 심볼, 관계, 설명 요약을 인덱싱하는 앱입니다.

2. 앱이 처리할 수 있는 정보

앱은 기기 내에서 아래 범주의 정보를 처리할 수 있습니다.

  • 사용자가 선택한 프로젝트 파일과 폴더 구조
  • 생성된 인덱스 요약, 심볼 설명, 관계 메모
  • 사용자 질문과 대화 기록
  • 로컬 또는 원격 LLM 서비스 연결에 필요한 프로바이더 설정

3. 로컬 우선 동작

KaninDex는 로컬 우선 흐름을 기준으로 설계됩니다. 로컬 인덱싱, 캐시된 프로젝트 구조, 기본 탐색 기능은 원격 서비스 전송 없이도 동작할 수 있습니다.

4. 원격 처리

사용자가 원격 LLM 프로바이더를 선택하면 질문 응답이나 설명 생성을 위해 프로젝트 맥락 일부가 해당 프로바이더로 전송될 수 있습니다. 여기에는 아래 정보가 포함될 수 있습니다.

  • 사용자 프롬프트
  • 프로젝트 요약
  • 관련 파일, 심볼, 관계 맥락
  • 응답 생성에 필요한 코드 발췌

KaninDex는 이 원격 처리 경로를 사용하기 전에 동의 단계를 제공합니다.

5. 자격 증명 및 저장 정보

프로바이더 설정, 프로젝트 인덱스, 대화 기록은 기기 로컬에 저장될 수 있습니다. 비밀값과 OAuth 토큰은 가능한 경우 암호화 형태로 저장되도록 설계됩니다.

6. 사용자 통제

사용자는 다음을 선택할 수 있습니다.

  • 원격 프로바이더 사용 여부
  • 원격 처리 동의 보류
  • 저장된 프로젝트와 로컬 대화 기록 삭제

7. 문의

개인정보 처리 관련 문의는 shteosis@gmail.com 으로 보낼 수 있습니다.
